Suno Video Generator

AI-powered lip-sync video generation — feed it a portrait and audio, get a realistic talking-head video out.

Python PyTorch FastAPI Gradio Whisper VAE FFmpeg

Portrait + audio in → AI lip-sync video out.

What it is

A real-time AI video generation tool built on MuseTalk that creates lip-synced talking videos from a single portrait image and an audio track. You give it a photo and some audio — a Suno-generated song, a voice recording, whatever — and it produces a video where the face naturally mouths along to the audio.

It runs locally on your GPU (NVIDIA), generating 30+ fps on a V100. The model uses a single-step inpainting approach (not iterative diffusion), which is what makes it fast enough for real-time use.

Key Features

  • Single-shot lip sync — One portrait image + any audio = realistic talking-head video. No training required per face.
  • Real-time generation — 30+ fps on NVIDIA GPU using single-step inpainting, not iterative diffusion.
  • Multi-language support — Works with English, Chinese, and Japanese audio out of the box.
  • Fine-grained control — Adjustable bounding box, cheek width, jaw margin, and face parsing mode to tune the output.
  • Debug mode — Test inpainting on the first frame before committing to a full render.
  • Dual interface — Gradio web UI for interactive use, FastAPI endpoint for programmatic access.

How it works

The pipeline chains together several ML models. First, Whisper-Tiny encodes the audio into feature embeddings that capture mouth shape information. The reference portrait goes through a VAE encoder to get a latent representation. A face parsing model identifies the lip region to create an inpainting mask.

Then a custom UNet (architecturally similar to Stable Diffusion's, but repurposed for single-step inpainting) takes the masked latent, the audio embedding, and generates the lip region in a single forward pass — no iterative denoising. The result gets decoded through the VAE decoder, composited back onto the original frame, and rendered out as video with FFmpeg.

Input Portrait image + audio file (MP3/WAV)
↓
Encode Whisper-Tiny (audio → embeddings) + VAE (image → latent space)
↓
Parse Face parsing model → lip region mask + bounding box
↓
Generate Custom UNet single-step inpainting → lip-synced latent
↓
Decode VAE decoder → composite onto original frame → FFmpeg render

The Gradio UI

The interface is split into two columns. On the left: upload your audio and reference portrait, then dial in the parameters — bounding box shift, jaw margin, cheek width, and parsing mode. There's a "Test Inpainting" button to preview the first frame before committing to a full render. On the right: the debug preview and the final output video player.

For programmatic use, the FastAPI backend exposes a POST /generate endpoint that accepts the same inputs and returns the rendered video, making it easy to integrate into other pipelines.

Why I built it

I wanted to generate music videos from Suno tracks without filming anything. The idea was simple: take a Suno-generated song, pair it with a portrait, and get a video where the face sings along. MuseTalk's single-step approach was the right fit because it's fast enough to iterate quickly — you can test different portraits, adjust the parameters, and preview without waiting minutes for each render.
